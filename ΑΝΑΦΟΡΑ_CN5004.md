## 1. Εισαγωγή και Περιγραφή Συστήματος

Η παρούσα εργασία υλοποιεί το **Σύστημα Διαχείρισης Κλινικής και Ραντεβού** (Εργασία 2), μια πλήρως λειτουργική desktop εφαρμογή σε **Java 21** με **JavaFX 21** για το γραφικό περιβάλλον. Η εφαρμογή προσομοιώνει το πληροφοριακό σύστημα μιας ιδιωτικής κλινικής, επιτρέποντας τη διαχείριση ιατρών, ασθενών και ραντεβού μέσω φιλικής γραφικής διεπαφής.

**Στόχος:** Ανάπτυξη εφαρμογής που αξιοποιεί αντικειμενοστραφή σχεδίαση (OOP), αποθήκευση δεδομένων σε αρχεία (File I/O) και γραφική διεπαφή με JavaFX, ικανοποιώντας πλήρως τις απαιτήσεις της εκφώνησης.

## 2. Λειτουργίες του Συστήματος

### 2.1 Πίνακας Ελέγχου (Dashboard)
Η κεντρική οθόνη εμφανίζει συνοπτικές στατιστικές σε μορφή καρτών:
- Συνολικός αριθμός ιατρών, ασθενών και ραντεβού
- Κατανομή ραντεβού ανά κατάσταση (Προγραμματισμένα / Ολοκληρωμένα / Ακυρωμένα)
- Λίστα με τους 5 πιο δραστήριους ιατρούς
- Πρόσφατα προγραμματισμένα ραντεβού

### 2.2 Διαχείριση Ιατρών
- **Προσθήκη** νέου ιατρού (Όνομα, Ειδικότητα, Τηλέφωνο, Email)
- **Ενημέρωση** στοιχείων υπάρχοντος ιατρού
- **Διαγραφή** ιατρού (με έλεγχο: δεν επιτρέπεται αν έχει ενεργά ραντεβού)
- **Αναζήτηση** σε πραγματικό χρόνο με βάση όνομα ή ειδικότητα

### 2.3 Διαχείριση Ασθενών
- **Εγγραφή** νέου ασθενή (Όνομα, Ημ. Γέννησης, Φύλο, Τηλέφωνο, Email, Ιατρικό Ιστορικό)
- **Ενημέρωση** και **Διαγραφή** (με έλεγχο ενεργών ραντεβού)
- **Αναζήτηση** σε πραγματικό χρόνο

### 2.4 Διαχείριση Ραντεβού
- **Προγραμματισμός** ραντεβού με επιλογή ιατρού, ασθενή, ημερομηνίας και ώρας (30λεπτες χρονοθυρίδες)
- **Αποτροπή διπλοκράτησης:** Αν ο ιατρός έχει ήδη ραντεβού την ίδια ημερομηνία και ώρα, το σύστημα εμφανίζει σχετικό μήνυμα και δεν επιτρέπει την καταχώρηση
- **Αλλαγή κατάστασης:** Ολοκλήρωση ή Ακύρωση ραντεβού
- **Φιλτράρισμα** πίνακα ανά κατάσταση

### 2.5 Αναφορές
- Φιλτράρισμα ιστορικού ραντεβού ανά κατάσταση, ιατρό και ασθενή
- Εξαγωγή αναφοράς σε αρχείο `data/report.txt`

## 3. Αλγοριθμική Επίλυση Προβλημάτων

### 3.1 Αποτροπή Διπλοκράτησης
Κατά την προσθήκη ή ενημέρωση ραντεβού, εκτελείται ο ακόλουθος έλεγχος στο `DataStore`:

```
ΓΙΑ ΚΑΘΕ υπαρχον_ραντεβου ΣΤΟ σύνολο:
    ΑΝ υπαρχον_ραντεβου.ιατρος == νεο.ιατρος
    ΚΑΙ υπαρχον_ραντεβου.ημερομηνια == νεο.ημερομηνια
    ΚΑΙ υπαρχον_ραντεβου.ωρα == νεο.ωρα
    ΚΑΙ υπαρχον_ραντεβου.κατασταση != ΑΚΥΡΩΜΕΝΟ
    ΚΑΙ νεο.id != υπαρχον_ραντεβου.id   (εξαίρεση για ενημέρωση)
        => ΡΙΨΕ IllegalStateException με μήνυμα στον χρήστη
```

### 3.2 Επιμονή Δεδομένων (File I/O)
Κάθε φορά που γίνεται προσθήκη, ενημέρωση ή διαγραφή, το `DataStore` επανεγγράφει ολόκληρη τη συλλογή στο αντίστοιχο CSV αρχείο. Κατά την εκκίνηση, τα αρχεία διαβάζονται και φορτώνονται στη μνήμη. Ειδικός χαρακτήρας `&#44;` χρησιμοποιείται για escape κομμάτων μέσα σε πεδία.

### 3.3 Αυτόματη Αρίθμηση ID
Η κλάση `IdGenerator` χρησιμοποιεί `AtomicInteger` για thread-safe αυτόματη αρίθμηση. Κατά τη φόρτωση δεδομένων, ο μετρητής αρχικοποιείται στο μέγιστο υπάρχον ID ώστε να αποφεύγονται συγκρούσεις.

## 4. Αρχιτεκτονική OOP

### 4.1 Ιεραρχία Κλάσεων

```
Doctor  (βασική κλάση)
  └── Specialist  (κληρονομεί Doctor – υπο-ειδικότητα, εβδ. χωρητικότητα)

Patient  (αυτόνομη οντότητα)

Appointment  (συνδέει Patient με Doctor)
  └── Status  (enum: SCHEDULED, COMPLETED, CANCELLED)
```

### 4.2 Βασικές Αρχές OOP που Εφαρμόστηκαν

| Αρχή | Υλοποίηση |
|---|---|
| **Ενθυλάκωση** | Όλα τα πεδία `private`, πρόσβαση μόνο μέσω getters/setters |
| **Κληρονομικότητα** | `Specialist extends Doctor` |
| **Πολυμορφισμός** | `DataStore` αποθηκεύει `List<Doctor>` που περιέχει και `Specialist` objects |
| **Singleton** | `DataStore.getInstance()` – μοναδική κοινή παρουσία |
| **Enum** | `Appointment.Status` για type-safe καταστάσεις |

## 6. Οδηγός Χρήσης (User Guide)

### Εκκίνηση
```bash
cd ~/Documents/GitHub/ClinicMS
JAVA_HOME=/usr/lib/jvm/java-21-openjdk mvn javafx:run
```
Τα αρχεία δεδομένων (`data/doctors.csv`, `data/patients.csv`, `data/appointments.csv`) δημιουργούνται αυτόματα στον φάκελο `data/` κατά την πρώτη εκκίνηση.

### Βήμα 1 – Προσθήκη Ιατρού
1. Κλικ στο **«Ιατροί»** στην πλαϊνή μπάρα
2. Συμπλήρωση πεδίων (Όνομα και Ειδικότητα είναι υποχρεωτικά)
3. Κλικ **«Προσθήκη Ιατρού»**
4. Ο ιατρός εμφανίζεται άμεσα στον πίνακα και αποθηκεύεται στο `doctors.csv`

### Βήμα 2 – Εγγραφή Ασθενή
1. Κλικ στο **«Ασθενείς»**
2. Συμπλήρωση στοιχείων (Ημ. Γέννησης σε μορφή `ηη/ΜΜ/εεεε`)
3. Κλικ **«Εγγραφή Ασθενή»**

### Βήμα 3 – Προγραμματισμός Ραντεβού
1. Κλικ στο **«Ραντεβού»**
2. Επιλογή Ασθενή και Ιατρού από τις λίστες
3. Εισαγωγή Ημερομηνίας (`ηη/ΜΜ/εεεε`) και επιλογή Ωρας
4. Συμπλήρωση Αιτίας επίσκεψης
5. Κλικ **«Προγραμματισμός Ραντεβού»**
   - Αν ο ιατρός έχει ήδη ραντεβού εκείνη την ώρα, εμφανίζεται μήνυμα σφάλματος

### Βήμα 4 – Ολοκλήρωση / Ακύρωση Ραντεβού
1. Επιλογή ραντεβού από τον πίνακα
2. Κλικ **«Ολοκλήρωση»** ή **«Ακύρωση Ραντεβού»**

### Βήμα 5 – Αναφορές
1. Κλικ στο **«Αναφορές»**
2. Φιλτράρισμα με κατάσταση, ιατρό ή ασθενή
3. Κλικ **«Εξαγωγή σε αρχείο report.txt»** για αποθήκευση

### Ενημέρωση / Διαγραφή
- Κλικ σε οποιαδήποτε γραμμή του πίνακα → τα στοιχεία φορτώνουν αυτόματα στη φόρμα
- Τροποποίηση και κλικ **«Ενημέρωση»** ή **«Διαγραφή»**

## 10. Επιπλέον Χαρακτηριστικά

### 10.1 Validation
Το σύστημα επαληθεύει όλα τα πεδία εισόδου:
- Ημερομηνίες σε μορφή `ηη/ΜΜ/εεεε`
- Ώρες σε μορφή `ΩΩ:ΛΛ`
- Email με regex pattern
- Τηλέφωνο με regex pattern
- Υποχρεωτικά πεδία (όνομα, ειδικότητα κλπ.)

### 10.2 User Experience
- Άμεση ανατροφοδότηση με status labels
- Χρωματική κωδικοποίηση για καταστάσεις
- Αυτόματη φόρτωση δεδομένων στη φόρμα με κλικ στον πίνακα
- Αναζήτηση σε πραγματικό χρόνο
- Φιλτράρισμα με πολλαπλά κριτήρια

### 10.3 Error Handling
- Try-catch blocks σε όλες τις κρίσιμες λειτουργίες
- Μηνύματα σφάλματος με Alert dialogs
- Έλεγχος για διπλοκράτηση ραντεβού
- Έλεγχος για διαγραφή ιατρού/ασθενή με ενεργά ραντεβού

### 10.4 Data Persistence
- Αυτόματη αποθήκευση σε κάθε αλλαγή
- CSV format με escape characters για κόμματα
- Αυτόματη δημιουργία data directory
- Thread-safe ID generation
