# Σύστημα Διαχείρισης Κλινικής
### CN5004 · Προχωρημένος Προγραμματισμός · Ατομική Εργασία 2025–26

Μια πλήρης **JavaFX desktop εφαρμογή** για τη διαχείριση ιατρών, ασθενών και ραντεβού ιδιωτικής κλινικής.

---

## Χαρακτηριστικά

| Ενότητα | Λειτουργικότητα |
|---|---|
| **Ιατροί** | Προσθήκη, ενημέρωση, διαγραφή, αναζήτηση με όνομα/ειδικότητα |
| **Ασθενείς** | Εγγραφή, ενημέρωση, διαγραφή, αναζήτηση· παρακολούθηση ιατρικού ιστορικού |
| **Ραντεβού** | Προγραμματισμός (με αποτροπή διπλοκράτησης), ενημέρωση, ολοκλήρωση, ακύρωση· φιλτράρισμα ανά κατάσταση |
| **File I/O** | Όλα τα δεδομένα αποθηκεύονται σε `data/*.csv` αρχεία σε κάθε αλλαγή |
| **Validation** | Έλεγχος ημερομηνίας (ηη/ΜΜ/εεεε), τηλεφώνου, email, υποχρεωτικών πεδίων σε κάθε φόρμα |

---

## Τεχνολογίες

- **Java 21**
- **JavaFX 21** (FXML + CSS)
- **Maven** (build & διαχείριση εξαρτήσεων)
- Αρχιτεκτονική OOP: κλάσεις μοντέλων `Doctor`, `Patient`, `Appointment`
- File I/O βασισμένο σε CSV μέσω singleton service `DataStore`

## Πώς να Εκτελέσετε

### Προαπαιτούμενα
- JDK 21+
- Maven 3.8+

### Βήματα

```bash
# 1. Μετάβαση στο project
cd ClinicMS

# 2. Build
mvn clean package

# 3. Εκτέλεση μέσω Maven JavaFX plugin
mvn javafx:run

# Εναλλακτικά: Εκτέλεση ως standalone JAR
java -jar target/ClinicMS-1.0-SNAPSHOT.jar
```

Ο φάκελος `data/` δημιουργείται αυτόματα κατά την πρώτη εκτέλεση.

## Σχεδιασμός OOP

- **Doctor** – id, name, specialty, phone, email
- **Patient** – id, name, dateOfBirth, gender, phone, email, medicalHistory
- **Appointment** – id, patientId, doctorId, date, timeSlot, reason, status (`SCHEDULED / COMPLETED / CANCELLED`), notes
- **DataStore** (Singleton) – λίστες στη μνήμη + επιμονή CSV· επιβολή αποτροπής διπλοκράτησης
- **IdGenerator** – thread-safe αυτόματη αρίθμηση ανά τύπο οντότητας
- **Validator** – στατικοί helpers για ημερομηνία, ώρα, email, τηλέφωνο

## Τεχνική Υλοποίηση

### OOP Αρχές
- **Encapsulation**: Όλα τα πεδία private με getters/setters
- **Inheritance**: `Specialist extends Doctor`
- **Polymorphism**: `List<Doctor>` περιέχει και `Specialist` objects
- **Singleton**: `DataStore.getInstance()`
- **Enum**: `Appointment.Status` για type-safe καταστάσεις

### Validation
- Ημερομηνίες: `ηη/ΜΜ/εεεε`
- Ώρες: `ΩΩ:ΛΛ`
- Email: regex pattern
- Τηλέφωνο: regex pattern
- Υποχρεωτικά πεδία

### Error Handling
- Try-catch blocks σε όλες τις κρίσιμες λειτουργίες
- Alert dialogs για μηνύματα σφάλματος
- Έλεγχος διπλοκράτησης
- Έλεγχος ενεργών ραντεβού πριν τη διαγραφή

### File I/O
- CSV format με escape characters
- Αυτόματη αποθήκευση σε κάθε αλλαγή
- Thread-safe ID generation
- Αυτόματη δημιουργία data directory

## Οδηγίες Χρήσης

### 1. Προσθήκη Ιατρού
- Κλικ στο **«Ιατροί»** στην πλαϊνή μπάρα
- Συμπλήρωση πεδίων (Όνομα και Ειδικότητα υποχρεωτικά)
- Κλικ **«Προσθήκη Ιατρού»**

### 2. Εγγραφή Ασθενή
- Κλικ στο **«Ασθενείς»**
- Συμπλήρωση στοιχείων (Ημ. Γέννησης: `ηη/ΜΜ/εεεε`)
- Κλικ **«Εγγραφή Ασθενή»**

### 3. Προγραμματισμός Ραντεβού
- Κλικ στο **«Ραντεβού»**
- Επιλογή Ασθενή και Ιατρού
- Εισαγωγή Ημερομηνίας και Ώρας
- Κλικ **«Προγραμματισμός Ραντεβού»**

### 4. Ολοκλήρωση/Ακύρωση
- Επιλογή ραντεβού από τον πίνακα
- Κλικ **«Ολοκλήρωση»** ή **«Ακύρωση»**

### 5. Αναφορές
- Κλικ στο **«Αναφορές»**
- Φιλτράρισμα με κατάσταση/ιατρό/ασθενή
- Κλικ **«Εξαγωγή σε αρχείο»**
